# study-plan skill 改进设计 v2

日期：2026-03-11

## 背景

用户在使用 study-plan skill 生成 rbuf 内存模型学习计划后，发现 3 个问题：

1. **调用链源码链接只标单行**：`03-调用链.md` 中函数实现的源码位置只给了单行号（如 `L3590`），但函数实现不可能只有一行
2. **源码链接显示文本不够直观**：链接文本只有文件名，行号范围在链接外部，不如 `[文件名:起始行-结束行](路径)` 格式直观
3. **核心函数缺少源码和解释**：`02-核心函数.md` 只有函数签名和文字描述，没有完整的源码实现代码块，也没有对源码的逐段解释

## 修改方案

### A. 修改 `format-rules.md` — 源码链接格式

**修改位置**：`## 链接格式` 中的源码位置模板

**变化**：
- 行号范围从「链接外部」移到「链接文本内部」
- 格式从 `[文件名](路径#L起始行) L起始-L结束` 改为 `[文件名:起始行-结束行](路径#L起始行)`
- 新增规则：行号范围必须反映完整实现范围，不允许只标单行
- 新增表格例外：表格内只写文件名，不需要行号范围

**影响范围**：所有 study-* skill 统一受益。

### B. 修改 `SKILL.md` 第 5.5 节 — 调用链源码范围

**修改位置**：`#### 5.5 生成 03-调用链.md`

**变化**：
- 明确要求调用层次详解中引用函数时必须给出源码位置
- 行号范围必须覆盖函数的完整实现，不允许只标注函数声明的单行

### C. 修改 `SKILL.md` 第 5.4 节 — 核心函数完整源码 + 逐段注释

**修改位置**：`#### 5.4 生成 02-核心函数.md`

**变化**：
- 原「函数签名（一行）」→ 改为「源码实现（完整原始 C 代码块，不省略不重写）」
- 原「关键逻辑步骤（纯文��）」→ 改为「逐段解释」，每段标注行号范围
- 每个函数详解结构调整为：标题 → 源码位置 → 一句话描述 → 源码实现 → 逐段解释 → 参数表 → 调用关系

**逐段解释格式**：
```
1. **第 X-Y 行：阶段名称** — 解释这段代码做什么、为什么这样做
2. **第 Z-W 行：阶段名称** — ...
```
