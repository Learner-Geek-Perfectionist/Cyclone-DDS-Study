### 步骤 3：实现主题分析逻辑（LSP 优先）

**任务**：
- 实现代码元素定位
- 实现调用链分析
- 实现数据结构识别

**实现内容**：

```markdown
### 2. 分析主题

#### 2.1 定位代码元素

利用 LSP 代码智能：
- 使用 Grep 在源码路径中搜索主题关键词，定位相关文件
- 使用 Read 工具读取相关文件，识别函数和结构体定义
- 提取函数签名、结构体定义的精确行号

降级策略：
- 优先使用 Grep 的精确模式搜索函数定义（如 `^func_name\s*\(`）
- 对于结构体，搜索 `struct struct_name` 或 `typedef struct`

#### 2.2 构建初步清单

收集以下信息：
- 相关文件列表（文件路径 + 说明）
- 相关结构体列表（名称 + 定义位置 + 行号范围）
- 相关函数列表（名称 + 定义位置 + 行号范围）

#### 2.3 扩展分析

对于每个核心函数：
- 使用 Read 读取函数实现
- 分析函数内部调用的其他函数（通过代码模式识别）
- 识别函数使用的数据结构（通过变量声明和类型识别）
- 构建调用关系（调用者 → 被调用者）

对于每个核心结构体：
- 使用 Grep 查找使用该结构体的函数
- 识别结构体之间的包含关系（通过字段类型分析）
```

**验收标准**：
- 能够定位主题相关的文件、函数、结构体
- 能够提取精确的行号信息
- 能够构建基本的调用关系

### 4. 深度分析

#### 4.1 分析数据结构

对于每个结构体：
- 使用 Read 读取定义所在文件

## 关键实现要点

### 代码分析策略

由于 Claude Code CLI 环境中 LSP 功能有限，实际实现采用以下策略：

1. **函数定位**：使用 Grep 精确模式搜索函数定义
   - C 语言：`^return_type\s+func_name\s*\(`
   - 提取行号用于超链接

2. **结构体定位**：使用 Grep 搜索结构体定义
   - 模式：`struct\s+struct_name` 或 `typedef struct`

3. **调用关系分析**：通过 Read 读取函数实现，使用代码模式识别
   - 识别函数调用：`func_name\s*\(`
   - 构建调用图

4. **行号范围确定**：
   - 读取定义所在文件
   - 通过代码块识别（大括号匹配）确定结束行

### 超链接生成规范

```markdown
[符号名](file:///绝对路径#L行号)（阅读 L起始-L结束）
```

**获取绝对路径**：
- 使用 `pwd` 命令获取当前工作目录
- 拼接相对路径得到绝对路径

### Mermaid 图生成技巧

1. **节点数量控制**：最多 20 个节点
2. **层次深度控制**：调用链深度 3-5 层
3. **样式应用**：
   - 入口函数：`style A fill:#e1f5ff`
   - 数据结构：`style F fill:#fff4e1`

### 错误处理

- Grep 未找到结果：记录并跳过
- 文件读取失败：记录并继续
- 规范文件不存在：跳过 04-规范对应.md
