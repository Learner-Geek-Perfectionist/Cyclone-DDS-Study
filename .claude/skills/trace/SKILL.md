# 源码追踪

用户提供函数名、文件路径或数据流程名称时，按以下步骤执行：

## 流程

1. **定位源码**：根据用户指定的函数/路径，找到对应的源码文件
   - 源码根目录: `cyclonedds/src/core/`
   - 头文件: `cyclonedds/src/core/ddsi/include/dds/ddsi/`

2. **阅读并解释**：
   - 关键数据结构：列出结构体字段含义
   - 核心函数：解释函数签名、参数、返回值
   - 内部逻辑：逐步解析关键代码路径

3. **追踪调用链**：
   - 从入口函数向下追踪调用关系
   - 格式: `函数A() -> 函数B() -> 函数C()`
   - 每个函数附 `文件名:行号`

4. **关联规范**：
   - 将源码实现关联回 RTPS/DDS/XTypes 规范中的对应概念
   - 指出实现与规范的差异（如有）

5. **引用已学知识**：
   - 读取 `progress/tracker.md`，关联用户已掌握的相关概念
   - 用已学概念做类比，避免重复讲解

## 输出格式

使用简体中文。每个函数/结构体的解释附源码出处 (`文件:行号`)。
调用链使用文本缩进或箭头格式清晰展示层级关系。
