## 📚 学习主题

### 主题 1：rbuf 系列函数的层次关系

#### 核心概念

1. **ddsi_rbuf_alloc_new**（`ddsi_radmin.c:427`）：纯 malloc 构造器，分配 `sizeof(ddsi_rbuf) + rbuf_size` 的内存，初始化字段，不修改 `rbp->current`
2. **ddsi_rbuf_new**（`ddsi_radmin.c:448`）：上层封装，调用 `alloc_new` + `release(旧current)` + 替换 `rbp->current`，加锁保护
3. **ddsi_rbuf_alloc**（`ddsi_radmin.c:491`）：从当前 rbuf 的 freeptr 处 bump 分配一块 `max_rmsg_size` 大小的内存，空间不足时调用 `rbuf_new` 换新 rbuf
4. **ddsi_rbuf_release**（`ddsi_radmin.c:463`）：原子递减 `n_live_rmsg_chunks`，归零则 `ddsrt_free` 整个 rbuf

#### 调用链

```
ddsi_rbufpool_new → ddsi_rbuf_alloc_new (初始化第一个 rbuf)

ddsi_rmsg_new → ddsi_rbuf_alloc → [空间不足?] → ddsi_rbuf_new → ddsi_rbuf_alloc_new + ddsi_rbuf_release

ddsi_rmsg_alloc → [chunk满?] → ddsi_rbuf_alloc → [同上]

ddsi_rmsg_free → foreach chunk: ddsi_rbuf_release
```

#### 关键设计决策

- **alloc_new vs new 分离原因**：初始化时 `rbp->current == NULL`，不能调 `new`（内含 `assert(rbp->current)`），只能直接调 `alloc_new`。运行期必须通过 `new` 来 release 旧 rbuf + 替换 current
- **rbufpool 始终只有一个**：`rbp->current` 在不同时刻指向不同 rbuf，不是多个 rbp

### 主题 2：chunk 链表设计的必要性

#### 为什么不能用其他方案

| 方案 | 问题 |
|------|------|
| 预分配大块（无 chunk） | 每个 rmsg 预留空间要按最坏情况，rbuf 尾部浪费严重 |
| realloc 扩容 | bump allocator 没有 realloc；即使有，搬移内存导致所有 zoff/指针失效 |
| chunk 链表（当前方案） | 旧数据不动，新数据接在别处，按需扩容 |

#### chunk 的实际使用

- 初始 chunk 存放：UDP 原始包 + receiver_state + sampleinfo 等
- 新 chunk 只存放溢出的元数据，不存放 UDP 包数据
- 通过 `ddsi_rmsg_alloc` 追加的结构体：receiver_state、rdata、sampleinfo、defrag_iv、rsample、rsample_chain_elem
- 这些都是几十字节的小结构，128KB 的 chunk 通常绰绰有余
- **绝大多数 rmsg 只需 1 个 chunk**

### 主题 3：预留检查 vs 实际占用

#### 关键代码

- `ddsi_rbuf_alloc`（`ddsi_radmin.c:494`）：检查剩余空间 >= `max_rmsg_size_w_hdr`（128KB + 头部）
- `commit_rmsg_chunk`（`ddsi_radmin.c:600`）：`freeptr = chunk+1 + chunk.u.size`，只推进实际用量
- 小包可能只用 2KB，但检查时预留 128KB，commit 时只消耗 2KB

### 主题 4：默认配置值确认

| 参数 | 默认值 | 来源 |
|------|--------|------|
| `rbuf_size` | 1MB (1048576) | `defconfig.c:26` |
| `rmsg_chunk_size` | 128KB (131072) | `defconfig.c:27` |

注意：代码注释中提到 "64kB max_rmsg_size always suffices for a UDP packet"（`ddsi_radmin.c:321`），这是说 64KB 是下限（足以容纳任何 UDP 包），不是默认值。默认值是 128KB，多出的空间给 chunk 内追加元数据留余量。

## 💡 心得体会

- 函数命名容易混淆时，抓住关键区别：带 `new` 的创建 rbuf 本身，不带 `new` 的操作 rbuf 内部；`alloc_new` 只管生，`new` 管生也管换
- chunk 链表本质是在 bump allocator 上实现"不搬移扩容"的唯一方案
- 注释中的数值不等于实际配置值，必须查 defconfig.c 或初始化代码确认
- 已在 .cursorrules 中添加"事实准确性"规则，要求所有数值必须附源码出处
