## 📚 学习主题

### 主题 1：ddsi_rbuf_alloc 与 max_rmsg_size_w_hdr

#### 核心概念

1. **ddsi_rbuf_alloc**：从 rbufpool 当前 rbuf 上 bump 分配一块内存。空间够直接返回 freeptr；不够则分配新 rbuf 替换 current。返回即预留，commit 时才确认。
2. **max_rmsg_size_w_hdr**：计算 `max(rmsg头, chunk头) + max_rmsg_size`。两种场景取最坏情况，简化代码。`offsetof(rmsg, chunk) + sizeof(chunk)` 写法明确表达"rmsg 自身字段 + 内嵌 chunk 头"的结构。
3. **预留-确认两阶段协议**：单线程保证下，返回 freeptr 就等于分配完成。commit 时 refcount==0 则 freeptr 不动，实现零成本丢弃。

#### 理解要点

- `offsetof(rmsg, chunk) + sizeof(chunk) == sizeof(rmsg)` 由 DDSRT_STATIC_ASSERT 保证
- 同一时刻只有一个线程操作一个 rbufpool，不需要加锁
- freeptr 不动 = 下次分配覆盖旧数据 = 零成本丢弃

### 主题 2：ddsi_rmsg_alloc 的扩容路径

#### 核心概念

1. **正常路径**：`(chunk + 1) + chunk->u.size` 定位空闲起始，`u.size += size8P` 推进已使用量
2. **扩容路径**：chunk 满时 commit 旧 chunk（推进 freeptr），从 rbufpool 分配新 chunk，链入 chunk 链表
3. **与 ddsi_rbuf_alloc 对比**：ddsi_rmsg_alloc 立即推进 `chunk->u.size`（同一 chunk 内多次分配），ddsi_rbuf_alloc 延迟推进 freeptr（单线程一次一个 rmsg）

### 主题 3：内存释放完整链路

#### 核心概念

1. **释放链路**：`rdata_unref → rmsg_unref(atomic_dec) → rmsg_free → rbuf_release(atomic_dec n_live_rmsg_chunks) → ddsrt_free(rbuf)`
2. **ddsrt_free**：libc `free()` 的薄封装，加 NULL 检查，支持通过 `ddsrt_set_allocator()` 替换分配器
3. **分片数据不合并**：fragchain 零拷贝引用各自 rmsg 中的数据，不做大拷贝
4. **rmsg 无独立 malloc**：直接在 rbuf 的 raw[] 中就地构造，rbuf 被 free 时所有内容一起释放

#### 理解要点

- 释放粒度是整个 rbuf，不是单个 rmsg
- rbuf 上只要还有一个 rmsg chunk 活着，整个 rbuf 就不会被释放
- rdata->rmsg（back-pointer）是引用计数管理的关键：rdata 本身无独立 refcount

### 主题 4：defrag fragtree 数据结构

#### 核心概念

1. **defrag_iv 结构体**：AVL 树节点，`[min, maxp1)` 表示一段连续已收到的字节范围，`first/last` 链接覆盖该范围的 rdata 链表
2. **fragtree**：以 `min` 为排序键的 AVL 树，每个节点代表一段连续区间
3. **sentinel 哨兵**：`[0, 0)` 节点，简化边界处理，保证 `avl_lookup_pred_eq` 总有前驱
4. **区间合并**：相邻/重叠区间合并为更大区间，树节点数递减
5. **重组完成判定**：树合并为单节点 `[0, sample_size)` 时完成
6. **lastfrag 优化**：指向 min 最大的区间节点，分片顺序到达时 O(1) 快速路径

### 主题 5：defrag_rsample_addiv 函数

#### 核心概念

1. 从 rmsg chunk 中分配 defrag_iv 节点
2. 用 rdata 的 `[min, maxp1)` 初始化区间
3. ddsi_rdata_addbias 保护 rmsg 生命周期（插入树 = 加 BIAS，取出时去 BIAS）
4. 插入 fragtree，利用调用者传入的 path 避免重复查找
5. 更新 lastfrag 快捷指针

## 💡 心得体会

- bump allocator 的"预留-确认"两阶段设计非常优雅：分配是返回指针，丢弃是什么都不做，确认是推进指针
- 内存释放链路贯穿了整个 radmin 模块的设计：rdata 无独立 refcount，全部寄生在 rmsg 上，rbuf 整体释放
- fragtree 的区间合并设计和 lastfrag 快速路径体现了对常见场景（顺序到达）的优化思路
- static 函数不带 ddsi_ 前缀是 Cyclone DDS 的命名约定，之前看到时有些困惑，现在明确了
